Technical

The project is open source and free to use. It uses the Google Maps API - Distance Radius Matrix for determing distances between appointments and map generation. This is currently capped to the free version for non-commercial use. It also uses getaddress.io for address look-ups using a postcode. Again this is limited to the free version. Both have daily quotas. If this is insufficient for your purposes, then consider upgrading each package; neither of which are expensive. All the source code is available on GitHub at https://github.com/DystopianProgrammer

The demo version of this application is currently deployed on DigitalOcean within two docker containers. The project is provided with docker files for ease of use and easy deployment. With docker, you can have the application up and running within your own environment in a matter of minutes. If you require assistance with this, please contact me for additional support, though reasonable fees will apply.

Configuration

The configuration is done through a yaml file which is located in the emilena-docker repository: https://github.com/DystopianProgrammer/emilena-docker

Within this project there are 3x directories.

  • api
  • db
  • web

The api directory contains a shell script which takes a single parameter which points to the dropwizard project. It then downloads and installs any dependencies and then starts up a docker container. emilena.yaml provides all the necessary configuration for the application. For example the SQL Dialect, and all application settings under 'applicationConfiguration'

The db directory just provides a DDL for the database. This is generated from a PostgreSQL export.

The web directory, similar to the api directory, also provides a shell script which takes a single parameter pointing to the front end web project.

To run each script, simply type ./run.sh /location/to/deployable/project

To build each deployable, from the emilena-api project, ensure you have Apache Maven installed, and run mvn clean package. To build the emilena-rota front-end, ensure you have ng cli installed using npm. First install all dependencies by typing npm install. Once that is done, then from the same project, type ng build --prod

The front-end is deployed in an Nginx HTTP server, and the configuration file for this is located in the same docker directory. However the application could be deployed with any HTTP server including over HTTPS. Proxying is required to prevent CORS.
